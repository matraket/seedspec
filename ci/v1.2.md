### RNF Base Guidelines (Doc 2 - NEXT)
Categorías agnósticas de tecnología:
- **Seguridad**: Autenticación, autorización, cifrado, auditoría
- **Rendimiento**: Tiempos respuesta, concurrencia, escalabilidad
- **RGPD/Cumplimiento**: Consentimientos, derechos ARCO, retención datos
- **Disponibilidad**: SLAs, backups, recuperación
- **Usabilidad**: Accesibilidad, responsive, i18n
- **Mantenibilidad**: Documentación, testing, logging

Debe referenciar RFs de KB-003 donde aplique (ej: N11RF01-N11RF17 para RGPD).

### DDD Guidelines (Doc 3)
- Bounded Contexts derivados de KB-001 + agrupación de RFs por dominio
- Para cada BC: Aggregates, Entities, Value Objects, Domain Events
- Context Map con relaciones entre BCs
- Mapeo BC ↔ RFs

### MVP Scope (Doc 7)
Priorizar según KB-001: "2-3 bounded contexts core"
- Core: Membresía (N3), Tesorería (N4)
- Supporting: Eventos simplificado (N5 parcial)
- Excluir v1: N12 (específicas), N13 (aragonés)
